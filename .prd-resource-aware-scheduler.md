## PRD - Resource Aware Scheduler (前端部分)

**需求来源**: Brain 自动调度，Design doc: /home/xx/dev/cecelia-core/docs/RESOURCE_AWARE_SCHEDULER.md

**功能描述**:
在 CeceliaOverview 页面添加双服务器集群状态显示，调用 `/api/brain/cluster/status` API 展示：
1. US VPS 和 HK VPS 的实时资源状态（CPU、内存、slots）
2. 集群整体状态（healthy/partial/degraded）
3. 各服务器运行中的任务列表
4. 资源告警指示（颜色编码）

**依赖关系**:
- 后端 API `GET /api/brain/cluster/status` 需要先在 cecelia-core 实现
- 如果 API 不存在，前端应该优雅降级（显示"单服务器模式"）

**涉及文件**:
- `apps/core/features/execution/pages/CeceliaOverview.tsx` - 添加 ClusterStatus 组件
- `apps/core/features/execution/components/ClusterStatus.tsx` - 新组件
- `apps/core/features/execution/api/agents.api.ts` - 添加 fetchClusterStatus API

**成功标准**:
- [ ] ClusterStatus 组件能够显示双服务器状态
- [ ] 当 API 不可用时优雅降级
- [ ] 与设计文档中的 UI 布局一致
- [ ] 每 30 秒自动刷新数据

**非目标**:
- 后端 API 实现（在 cecelia-core 另外做）
- HK Bridge 部署
- 动态席位计算逻辑
