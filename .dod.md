# DoD - 扩展 intent.js phrase patterns 覆盖率

QA: docs/QA-DECISION.md

## 验收标准

### 功能验收

- [ ] CREATE_GOAL 至少 15 个 patterns (当前 7 个)
      验证方法: 检查 nlp-parser.ts INTENT_PATTERNS.CREATE_GOAL 数组长度 >= 15
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] CREATE_PROJECT 至少 15 个 patterns (当前 6 个)
      验证方法: 检查 nlp-parser.ts INTENT_PATTERNS.CREATE_PROJECT 数组长度 >= 15
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] CREATE_TASK 至少 15 个 patterns (当前 13 个)
      验证方法: 检查 nlp-parser.ts INTENT_PATTERNS.CREATE_TASK 数组长度 >= 15
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] QUERY_TASKS 至少 15 个 patterns (当前 9 个)
      验证方法: 检查 nlp-parser.ts INTENT_PATTERNS.QUERY_TASKS 数组长度 >= 15
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] UPDATE_TASK 至少 15 个 patterns (当前 8 个)
      验证方法: 检查 nlp-parser.ts INTENT_PATTERNS.UPDATE_TASK 数组长度 >= 15
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] 每个意图类型至少包含 5 个中文口语化表达
      定义: 口语化表达 = 使用日常非正式动词/助词的中文 pattern
      示例关键词: "帮我"、"搞个"、"弄个"、"整个"、"加个"、"建个"、"来个"、"给我"等
      验证方法: 人工检查每个 INTENT_PATTERNS 数组，确认至少 5 个符合口语化特征的 pattern
      Test: manual:检查 nlp-parser.ts 中每个意图类型的 patterns，确认口语化表达数量
- [ ] 每个意图类型至少包含 3 个英文表达
      定义: 英文表达 = 纯英文 pattern (不含中文字符)
      验证方法: 遍历每个 INTENT_PATTERNS 数组，匹配纯英文 pattern，计数 >= 3
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] 运行 npm test -- intent-recognition.test.ts 全部通过
      Test: apps/core/src/__tests__/intent-recognition.test.ts
- [ ] 手动验证: "帮我创建一个任务叫做XX" 能正确识别为 CREATE_TASK
      Test: manual:启动 Core API, 调用 /api/intent/recognize 验证
