## PRD - 增强 Cecelia Overview 前端页面显示关键运行指标

**需求来源**: 用户要求在前端页面显示 Cecelia 系统运行的关键信息，包括座位配置、Tick Loop 状态、当前活动、熔断器状态和任务队列统计

**功能描述**:
增强 `/system/cecelia` 页面（CeceliaOverview.tsx），新增以下实时监控面板：

1. **Seats 配置区域**
   - 显示最大并发数（max_concurrent）
   - 显示当前使用中的 seats 数量
   - 显示可用 seats 数量

2. **Tick Loop 实时状态区域**
   - 是否运行中（enabled + loop_running）
   - 上次运行时间（last_tick）
   - 下次运行时间（next_tick）
   - 今日执行次数（actions_today）
   - 循环间隔（loop_interval_ms）

3. **当前活动区域**
   - 当前正在执行的任务（in_progress 任务列表）
   - 最后派发的任务（last_dispatched_task）

4. **熔断器状态区域**
   - 显示所有 circuit breaker 的状态（OPEN/CLOSED/HALF_OPEN）
   - 显示失败次数
   - 显示最后失败时间（如果有）

5. **任务队列统计区域**
   - queued 任务数量
   - in_progress 任务数量
   - completed 任务数量
   - failed 任务数量
   - cancelled 任务数量

**涉及文件**:
- `/home/xx/dev/cecelia-workspace/apps/core/features/execution/pages/CeceliaOverview.tsx` - 主要修改文件
- 可能需要新增 API 端点或使用现有端点获取任务统计

**数据来源**:
- Brain API: `http://localhost:5221/api/brain/tick/status` - 提供 Seats、Tick Loop、熔断器数据
- Tasks API: `/api/tasks/stats` 或现有端点 - 提供任务队列统计

**成功标准**:
1. 页面正确显示所有 5 个区域的实时数据
2. 数据自动刷新（例如每 30 秒或 1 分钟）
3. UI 清晰美观，使用卡片布局区分不同区域
4. 熔断器状态用颜色区分（CLOSED=绿色，OPEN=红色，HALF_OPEN=黄色）
5. Tick Loop 运行状态用图标或状态指示器显示
6. 任务队列统计用进度条或饼图可视化

**非目标**:
- 不修改 Brain API 的实现
- 不修改任务系统的核心逻辑
- 不添加任务操作功能（如取消、重试等）
- 不修改现有的执行历史展示逻辑
