# PRD: PARA Feature Layer

## 目标

为 Task Management System 添加 Feature 层级支持，实现完整的 PARA 结构。

## PARA 结构

```
Area (领域) - workspaces 表
  └── Project (项目 = Repository) - projects 表
        └── Feature (功能 = SubProject) - projects 表 (parent_id)
              └── Task (任务) - tasks 表
```

## 需求

### 1. 数据库更新

- [x] projects 表添加 `parent_id` 字段（已完成）

### 2. API 更新

- [ ] POST /api/tasks/projects 支持 parent_id
- [ ] GET /api/tasks/projects 支持查询子项目
- [ ] GET /api/tasks/projects/:id/children 获取子项目列表

### 3. 前端更新

- [ ] 项目列表显示层级结构
- [ ] 支持展开/收起子项目

## 验收标准

- 能创建 Feature 作为 Project 的子项
- 能查询 Project 的所有 Features
- 前端能显示 Project → Feature 层级
