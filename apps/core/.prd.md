## PRD - Cecelia 静默层 Phase 4：自驱运转

**需求来源**: Cecelia 自我复活计划（docs/CECELIA-REVIVAL-PRD.md Phase 4）

**功能描述**: 让 Cecelia 真正活起来：
1. 自驱模式 — tick 默认 enabled，并发 3，所有免疫机制确认就位
2. 自我进化 — 定期读取 EventLog 生成自我诊断报告 + 改进任务
3. 健康检查 API — 一个端点看到 Cecelia 全部器官状态

**涉及文件**:
- 新建 `src/brain/self-diagnosis.js` — 自我诊断模块
- 修改 `src/brain/tick.js` — 默认 enabled + 集成诊断调度
- 修改 `src/brain/routes.js` — 添加健康检查 + 诊断 API
- 修改 `apps/core/.env.example` — 添加 CECELIA_TICK_ENABLED 环境变量

**成功标准**:
1. 服务启动后 tick 自动运行（CECELIA_TICK_ENABLED=true）
2. GET /api/brain/health 返回所有器官状态
3. POST /api/brain/self-diagnosis 生成诊断报告
4. 诊断报告包含成功率、失败率、改进建议
5. brain/ 目录下所有测试通过

**非目标**:
- 不实现 cron 定时诊断（手动触发或 tick 内定时）
- 不动业务代码
- 不改前端
