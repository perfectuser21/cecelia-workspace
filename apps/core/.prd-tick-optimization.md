# PRD: Tick 优化 + 事件驱动

## 背景

当前 Tick 机制存在问题：
1. 30 分钟间隔太慢
2. 纯轮询，任务完成后要等下一轮
3. TRD 分解器创建的任务没有强制关联 OKR

## 功能改动

### 1. Tick 间隔改为 5 分钟

```javascript
const TICK_INTERVAL_MINUTES = 5;  // 从 30 改成 5
```

### 2. 事件驱动触发

在 cecelia-run 完成任务后，立即触发下一个任务：

```
cecelia-run 完成
    ↓
标记 Task 为 completed
    ↓
立即调用 executeTick()
    ↓
启动下一个任务
```

修改 executor.js：
- `onTaskComplete()` 回调触发 `executeTick()`

### 3. TRD 分解器强制关联 goal_id

```javascript
// decomposeTRD() 参数校验
if (!goalId) {
  throw new Error('goal_id is required');
}
```

### 4. 简化 TRD 存储

- 删除 `trd_decompositions` 表的强依赖
- TRD 内容存入 Task 的 payload 字段（可选）
- 保留 API 但简化存储

## 成功标准

- [ ] TICK_INTERVAL_MINUTES = 5
- [ ] 任务完成后立即触发下一个
- [ ] TRD 分解必须指定 goal_id
- [ ] 现有功能不受影响
