## PRD - 清理 workspace 旧 Brain 代码，改用 semantic-brain API

**需求来源**: 架构重构 - Brain 已迁移到 cecelia-semantic-brain（独立服务，端口 5221），需清理 workspace 的旧代码避免维护混乱

**功能描述**:
删除 workspace 的 `apps/core/src/brain/` 目录（17 源文件 + 13 测试），修改 `apps/core/src/dashboard/routes.ts` 改用 semantic-brain API 调用。具体：
1. routes.ts 移除 `import { parseIntent, parseAndCreate, INTENT_TYPES } from '../brain/intent.js'`
2. 添加 API 客户端函数调用 `http://localhost:5221/intent/parse` 和 `/intent/create`
3. 在测试中 mock fetch 避免依赖真实服务
4. 删除整个 `apps/core/src/brain/` 目录

**涉及文件**:
- 修改: `apps/core/src/dashboard/routes.ts`
- 修改: `apps/core/src/dashboard/__tests__/routes.test.ts` (或相关测试文件)
- 删除: `apps/core/src/brain/` (整个目录)

**成功标准**:
1. routes.ts 改用 fetch 调用 semantic-brain API（`/intent/parse`, `/intent/create`）
2. INTENT_TYPES 常量在 routes.ts 中本地定义
3. 测试中 fetch 被 mock，测试通过（不依赖真实 semantic-brain 服务）
4. apps/core/src/brain/ 目录已删除
5. 无其他文件引用 brain 目录（grep 确认）
6. npm test 通过

**非目标**:
- 不修改 semantic-brain 服务本身
- 不改变现有 intent 识别的功能逻辑
- 不添加新的 intent 类型或 API 端点
