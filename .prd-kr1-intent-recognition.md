# PRD - KR1: 意图识别 - 自然语言→OKR/Project/Task

## 需求来源
Brain 自动调度任务，目标是推进 O1 的第一个关键结果（KR1），当前进度 0%。

## 功能描述
实现自然语言到结构化任务的意图识别功能，能够：
1. 解析用户的自然语言输入（如"我想优化登录流程"）
2. 识别意图类型（创建 OKR、创建 Project、创建 Task）
3. 提取关键信息（标题、描述、优先级、所属项目等）
4. 自动映射到 PARA 层级结构（Area → Project → Feature → Task）
5. 调用对应的 Brain API 创建实体

## 涉及文件
- `apps/core/src/lib/api/brain/intent-recognition.ts` - 意图识别核心逻辑
- `apps/core/src/lib/api/brain/routes.ts` - 新增 `/api/brain/parse-intent` 端点
- `apps/core/src/lib/api/brain/types.ts` - 类型定义（IntentType, ParsedIntent）
- `apps/core/src/lib/api/brain/actions.ts` - 集成到 action 系统
- `apps/core/tests/api/brain/intent-recognition.test.ts` - 单元测试

## 成功标准
- [ ] API 端点 `POST /api/brain/parse-intent` 接受自然语言输入
- [ ] 能正确识别三种意图类型：`create-okr`、`create-project`、`create-task`
- [ ] 提取的信息包含：title、description、priority（如果提及）、parent（如果适用）
- [ ] 测试覆盖至少 5 种典型场景（创建目标、创建项目、创建任务、复杂描述、模糊输入）
- [ ] 返回结构化的 ParsedIntent 对象，包含 confidence 分数
- [ ] 对于低置信度结果（< 0.6），返回建议的追问问题

## 非目标
- ❌ 不实现多轮对话或上下文记忆（v1 只做单次解析）
- ❌ 不实现复杂的 NLP 模型训练（使用规则+简单启发式）
- ❌ 不处理批量意图识别（一次只处理一个输入）
- ❌ 不实现自动执行（只识别意图，不直接创建实体）
