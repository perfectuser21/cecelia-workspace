# PRD: Fix Quality Monitor Styling Isolation

## 问题

Quality Monitor 页面被父容器的样式影响，导致配色和其他实例（Autopilot/Core）混淆。

## 原因

- 其他 Core 页面使用 `-m-8 -mt-8` 抵消父容器 padding
- Quality Monitor 缺少这个，导致父容器背景色露出
- 页面无法完全控制自己的深色背景

## 解决方案

为 QualityMonitorPage 添加 `-m-8 -mt-8` class，确保：
- 完全独立的深色背景（from-slate-900 via-slate-800 to-slate-900）
- 不受 Core 或 Autopilot 主题颜色影响
- 保持 Quality Monitor 原有的独立配色风格

## 改动

- `apps/core/features/quality/pages/QualityMonitorPage.tsx`
  - 修改根 div: `min-h-screen -m-8 -mt-8 p-8 bg-gradient-to-br...`
