# PRD - 修复 platform-data Skill 软链接路径问题

## 需求来源

测试 skill-loader 时发现，通过软链接调用 command.sh 时路径解析错误。

## 问题描述

当前 command.sh 使用：
```bash
FEATURE_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
```

通过软链接 `~/.claude/skills/platform-data/command.sh` 调用时：
- `dirname` 返回软链接目录 `~/.claude/skills/platform-data`
- FEATURE_DIR 变成 `~/.claude/skills` 而不是真实的 Feature 目录

导致找不到 SKILL.md 文件。

## 解决方案

使用 `readlink -f` 解析真实路径：
```bash
SCRIPT_PATH="$(readlink -f "${BASH_SOURCE[0]}")"
FEATURE_DIR="$(cd "$(dirname "$SCRIPT_PATH")/.." && pwd)"
```

## 涉及文件

- `apps/features/platform-data/skill/command.sh` - 修复路径解析逻辑

## 成功标准

- ✅ 直接调用 command.sh 正常工作
- ✅ 通过软链接调用 command.sh 正常工作
- ✅ `bash ~/.claude/skills/platform-data/command.sh help` 显示帮助信息
