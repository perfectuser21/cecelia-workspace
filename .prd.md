## PRD - 添加开发/生产环境视觉标识

**需求来源**: 用户希望在浏览器中更容易区分开发环境（5212）和生产环境（5211）

**功能描述**:
为前端添加明显的环境标识，让用户打开页面一眼就知道当前是开发环境还是生产环境。

**实现方案**:
1. **顶部环境标签**：开发环境（5212）显示橙色横条 "🛠️ 开发环境 (DEV)"
2. **浏览器标题**：开发环境显示 "Cecelia [DEV]"，生产环境显示 "Cecelia"
3. **检测方式**：通过 `window.location.port` 判断（5212 = 开发，5211 = 生产）

**涉及文件**:
- `apps/dashboard/src/App.tsx` - 添加环境检测和顶部标签
- 在 `AppContent` 组件中添加环境标识逻辑

**技术要点**:
1. 使用 `useEffect` 设置浏览器标题
2. 开发环境添加固定定位的顶部横条
3. 横条样式：橙色背景、白色文字、z-index 最高
4. 顶部横条高度 48px（h-12），字体 text-lg，让标识更醒目
5. 主内容区需要相应调整 padding-top

**成功标准**:
1. 访问 http://perfect21:5212 时，顶部显示橙色 "🛠️ 开发环境 (DEV)" 标签
2. 访问 http://perfect21:5211 时，没有环境标签（正常显示）
3. 开发环境浏览器 Tab 显示 "Cecelia [DEV]"
4. 生产环境浏览器 Tab 显示 "Cecelia"
5. npm run build 成功编译
6. 两个环境的功能都正常工作

**非目标**:
- 不修改路由逻辑
- 不修改其他页面组件
- 不添加复杂的环境配置系统
