## PRD - Ops Dashboard + OKR Task Tree

**需求来源**: 用户要求实现两个核心前端视图，接入已有的 Brain API
**功能描述**:

### View 1: Ops Dashboard
- 新建 `OpsDashboard.tsx` 组件，作为 SystemTabbed 的第一个 tab "Ops"
- 调用 Brain API 端点: health, cluster/status, vps-slots, tick/status, alertness, circuit-breaker, token-usage, watchdog
- 双列 VPS 对比卡片 (US vs HK): CPU、Slots used/total、Brain status
- 系统状态条: Tick 运行状态、Alertness 等级、Token 今日消耗、断路器状态
- 活跃进程表 (watchdog 数据): TaskID, PID, CPU%, RSS(MB)
- 10s 轮询关键指标, 30s 轮询次要指标

### View 2: OKR Task Tree
- 新建 `OKRTaskTree.tsx` 组件，作为 WorkTabbed 的新 tab "Tasks"
- 调用: /api/tasks/goals, /api/tasks/tasks, /api/brain/quarantine, /api/brain/focus
- 树形结构: Objective → KR → Task (可展开/折叠)
- 统计条: Queued / In Progress / Done 计数
- 筛选: All / P0 / P1 / 按状态
- 隔离区 (Quarantine) 管理: 显示失败任务 + Release 按钮

**涉及文件**:
- 新建: `apps/core/features/system/pages/OpsDashboard.tsx`
- 新建: `apps/core/features/planning/pages/OKRTaskTree.tsx`
- 修改: `apps/core/features/system-hub/pages/SystemTabbed.tsx` (添加 Ops tab)
- 修改: `apps/core/features/work/pages/WorkTabbed.tsx` (添加 Tasks tab)
- 修改: `apps/core/features/work/index.ts` (添加 /work/tasks 路由, 移除 redirect)

**成功标准**:
1. `/system` 默认显示 Ops Dashboard tab，所有 API 数据正确渲染
2. `/work/tasks` 显示 OKR Task Tree，树形结构可展开折叠
3. 所有现有路由和 tab 不受影响
4. 遵循现有的设计风格 (dark mode, slate-800 cards, emerald/amber/red status colors)

**非目标**:
- 不修改后端 API
- 不添加新 npm 依赖
- 不修改其他已有页面的功能
