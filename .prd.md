# PRD: 调整 TaskDatabase 分页默认值

## 功能描述

调整 TaskDatabase 分页组件的默认配置：
- 默认每页显示数量从 50 个减少到 10 个
- 下拉选项从 20/50/100 调整为 10/20/50

## 背景

用户反馈默认 50 个任务过多，页面初次加载时仍有卡顿感。希望默认显示最近 10 个任务，保持页面轻量化。

## 改动文件

`apps/core/features/planning/pages/TaskDatabase.tsx`

两处修改：
1. Line ~173: `pageSize` 初始化默认值从 50 → 10
2. Line ~595: 下拉 `<option>` 从 20/50/100 → 10/20/50

## 成功标准

1. 刷新页面默认显示 10 个任务
2. 下拉菜单显示选项：10/页、20/页、50/页
3. localStorage 持久化功能正常（用户选择后下次打开保持）
4. npm run build 通过
