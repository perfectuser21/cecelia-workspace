## PRD - OKR Task Tree 交互编辑

**需求来源**: 用户在 /work/tasks 页面无法调整任务优先级和状态，当前为纯只读展示
**功能描述**:

为 OKRTaskTree 页面添加内联编辑功能：

1. **优先级编辑**: 点击任务/目标的 Priority 标签（P0/P1/P2），弹出下拉选择器，选择后调用 PATCH API 立即更新
2. **状态编辑**: 点击任务/目标的 Status 标签（queued/in_progress/completed/cancelled），弹出下拉选择器，选择后调用 PATCH API 立即更新
3. **乐观更新**: 选择后立即更新 UI，API 失败时回滚并提示错误

**涉及文件**:
- 修改: `apps/core/features/planning/pages/OKRTaskTree.tsx` — 添加 Priority/Status 点击编辑交互

**API 端点**:
- `PATCH /api/tasks/tasks/:id` — 更新任务 priority/status
- `PATCH /api/tasks/goals/:id` — 更新目标 priority/status

**成功标准**:
1. 点击 P0/P1/P2 标签弹出选择器，选择后自动保存
2. 点击状态标签弹出选择器，选择后自动保存
3. 更新失败时回滚 UI 并显示错误提示
4. vite build 通过，无 TypeScript 编译错误

**非目标**:
- 不添加拖拽排序功能
- 不添加任务创建/删除功能
- 不添加新 npm 依赖
- 不修改后端 API
