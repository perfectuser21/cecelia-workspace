# PRD: 简化 Work 路由，移除多余 tab

## 问题描述

Work 路由当前有 7 个 tab，结构混乱，包含了不应该在这里的管理页面。

**当前 tab 列表**：
1. Projects (/work)
2. OKR Hierarchy (/work/okr)
3. **Businesses** (/work/businesses) ← 不应该在这里
4. **Departments** (/work/departments) ← 不应该在这里
5. Tasks (/work/tasks)
6. Roadmap (/work/roadmap)
7. **Features** (/work/features) ← 应该移除

## 用户反馈

- "Work 里面应该是整体的 summary"
- "为什么 OKR 页面要关联到 business？点也点不开"
- "应该是 personal goal / global goal → business → department"
- "很乱"

## 解决方案

### 简化为 4 个核心 tab

保留核心功能，删除管理页面：

1. **Projects** (/work) - 项目总览
2. **OKR** (/work/okr) - OKR 层级视图
3. **Tasks** (/work/tasks) - 任务管理
4. **Roadmap** (/work/roadmap) - 路线图

### 删除的 tab

- ❌ **Businesses** - 这是管理页面，不是日常工作视图
- ❌ **Departments** - 这是管理页面，不是日常工作视图
- ❌ **Features** - 与 Projects 重复，且不够核心

## 修改文件

`apps/core/features/work/pages/WorkTabbed.tsx`

## 具体变更

1. 移除 `Building2, Users, Layers` 图标导入（不再需要）
2. 删除 Businesses tab 配置
3. 删除 Departments tab 配置
4. 删除 Features tab 配置
5. 只保留 4 个 tab：Projects, OKR, Tasks, Roadmap

## 验收标准

1. Work 页面只显示 4 个 tab
2. 删除的页面不再显示
3. 保留的 4 个 tab 正常工作
4. npm run build 通过
5. 前端刷新后显示正确

## 影响范围

- 前端路由变化：/work/businesses, /work/departments, /work/features 不再可访问
- 不影响这些页面的功能，只是不在 Work 路由中显示
- 如果未来需要访问 Businesses/Departments，可以在 System 或其他路由添加
