## PRD - OKR 系统重构为单层 Area OKR 架构

**需求来源**: 用户决定简化 OKR 层级结构，从多层 OKR 改为单层 Area OKR + 执行层分离

**功能描述**:
- 将 OKR 系统从 3 层（Area/Project/SubProject OKR）简化为 1 层（Area OKR only）
- 每个 Area 有一个 Objective + 3~5 个 Key Results
- Project/SubProject 作为执行层，通过 `contributes_to_kr_id` 关联到 KR
- 数据库新建 `key_results` 表存储 KR
- 修改 `goals` 表移除多层级字段（project_id, department_id, parent_id）
- 新建 OKR API 端点 `/api/okr/*`
- 重构前端页面：OKRDashboard（首页）+ AreaOKRDetail（Area 详情）

**涉及文件**:

**数据库**:
- `apps/core/src/db/migrations/XXX-create-key-results.sql` - 新建 key_results 表
- `apps/core/src/db/migrations/XXX-modify-goals-table.sql` - 修改 goals 表
- `apps/core/src/db/migrations/XXX-modify-projects-table.sql` - 修改 projects 表

**后端 API**:
- `apps/core/src/okr/index.js` - 新建 OKR API 路由
- `apps/core/src/okr/objectives.js` - Objectives CRUD
- `apps/core/src/okr/key-results.js` - Key Results CRUD
- `apps/core/src/index.js` - 注册 OKR 路由

**前端**:
- `apps/core/features/planning/pages/OKRDashboard.tsx` - 重构首页
- `apps/core/features/planning/pages/AreaOKRDetail.tsx` - 新建 Area 详情页
- `apps/core/features/planning/api/okr.api.ts` - 新建 OKR API 函数
- `apps/core/features/work/pages/WorkTabbed.tsx` - 更新路由引用
- `apps/core/features/work/index.ts` - 更新路由配置

**成功标准**:
1. 数据库 key_results 表创建成功
2. goals 表只保留 business_id，删除 project_id/department_id/parent_id
3. projects 表添加 contributes_to_kr_id 字段
4. API 端点 `/api/okr/areas`, `/api/okr/areas/:areaId`, `/api/okr/objectives/:id`, `/api/okr/key-results/:id` 正常工作
5. 前端 OKRDashboard 显示 Areas 卡片，点击进入 AreaOKRDetail
6. AreaOKRDetail 显示 Objective + KR 列表，KR 显示贡献的 Projects
7. 测试数据生成：ZenithJoy 2 个 O（每个 3-5 KR），Stock 1 个 O（3 KR）
8. 旧的 3 层测试数据清理完成
9. npm run build 成功
10. 前端访问 `/work/okr` 正常显示

**非目标**:
- 不做 OKR 编辑功能（交互页面后续实现）
- 不做 Project 到 KR 的自动进度同步（暂时手动设置）
- 不做 OKR 历史版本记录
- 不做 Cecelia 自动对齐逻辑（数据结构先稳定）
