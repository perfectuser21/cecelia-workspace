## PRD - 仓库清理：对齐代码与文件状态

**需求来源**: 用户要求扫描 repo，整理 feature 状态，清理垃圾使仓库将来能用起来

**功能描述**:
1. 删除根目录历史 PRD/DoD 文件（47 个），只保留当前 `.prd.md` 和 `.dod.md`
2. 删除废弃目录：`dashboard/`（已被 `apps/dashboard/frontend/` 取代）、`.git-rewrite/`、`apps/core/features/node_modules`
3. 清理 Feature 模块声明与文件不对齐的问题：
   - `planning/index.ts`：删除没有实现的页面路由声明（PlanningDashboard, AITeamOverview, CompanyOverview, MediaOverview, TeamOverview, FinanceOverview, ProjectsPage, PortfolioOverview）
   - `execution/index.ts`：删除 TaskMonitor 声明（文件不存在）
   - `system/index.ts`：删除 OpsOverview、SystemOverview 声明（文件不存在）
4. 删除其他垃圾文件：`.dev-mode.lock`、`.prd.md~develop`、`CeceliaOverview.tsx.backup`、gate 标记文件
5. 清理重复的 quality json 文件

**涉及文件**:
- 删除：47 个 `.prd-*.md` + `.dod-cp-*.md` 文件
- 删除：`dashboard/` 目录、`.git-rewrite/` 目录、`apps/core/features/node_modules/`
- 修改：`apps/core/features/planning/index.ts`（删除缺失页面的路由和组件声明）
- 修改：`apps/core/features/execution/index.ts`（删除 TaskMonitor 声明）
- 修改：`apps/core/features/system/index.ts`（删除 OpsOverview、SystemOverview 声明）
- 删除：`.dev-mode.lock`、`.prd.md~develop`、`apps/core/features/execution/pages/CeceliaOverview.tsx.backup`
- 删除：`.gate-*-passed` 文件、`.quality-report.json`（保留 `.quality-summary.json`）

**成功标准**:
1. 所有 Feature 模块的 index.ts 声明的组件，对应的文件都存在
2. 根目录没有历史 PRD/DoD 垃圾文件
3. 没有废弃目录（dashboard/、.git-rewrite/）
4. TypeScript 编译无报错
5. 现有功能不受影响（只删除不存在的声明和废弃文件）

**非目标**:
- 不创建缺失的页面文件（那是将来的 feature 工作）
- 不清理 git 分支（需要单独操作，影响远端）
- 不修改任何现有功能的逻辑代码
- 不动 `.archive/` 目录（已经是归档区）
