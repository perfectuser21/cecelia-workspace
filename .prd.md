# PRD: 修复 Orchestrator Proxy 冲突

## 问题

/api/orchestrator 的 proxy 配置覆盖了本地的 chat/action 路由，导致聊天功能不工作。

## 解决方案

将 proxy 改为只代理特定路径（health/state/query），让 chat/action 走本地路由。

## 成功标准

- [ ] /api/orchestrator/chat 返回正确响应
- [ ] /api/orchestrator/health 仍然代理到 Brain API
