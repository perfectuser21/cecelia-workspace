# PRD: 修复登录后跳转逻辑

## 问题描述
用户访问 `/features` 页面时未登录，被重定向到登录页。登录成功后自动跳转到首页 `/`，而不是用户原本想访问的 `/features` 页面。

## 预期行为
登录成功后应该跳转回用户原本想访问的页面，而不是硬编码跳转到首页。

## 解决方案

### 1. App.tsx 修改
在用户未登录时跳转到登录页时，将当前路径作为 `redirect` 参数传递：
```typescript
// 当前：返回 <FeishuLogin />
// 改为：navigate('/login?redirect=' + location.pathname)
```

### 2. FeishuLogin.tsx 修改
登录成功后，读取 `redirect` 参数并跳转回原页面：
```typescript
const redirect = searchParams.get('redirect') || '/';
navigate(redirect);
```

## 技术实现
- 使用 React Router 的 `useSearchParams` 读取 URL 参数
- 使用 `navigate()` 跳转

## 验收标准
- [ ] 访问 `/features` 未登录时，URL 变为 `/login?redirect=/features`
- [ ] 登录成功后自动跳转回 `/features`
- [ ] 没有 redirect 参数时默认跳转到首页 `/`
- [ ] 所有需要认证的路由都适用此逻辑
