# PRD: Staff 页面 Area → Department → Worker 三层显示

## 目标

重写 StaffPage.tsx，按 Area → Department → Worker 三层层级展示员工，匹配 workers.config.json 的 area/department 结构。

## 背景

- Brain API /staff 现在返回 areas 对象 + 每个 team 的 area/department 字段
- 当前 StaffPage.tsx 只显示平铺的 team 列表
- 用户期望看到：Cecelia（Area） → 核心团队（Department） → 具体员工

## 改动范围

### staffApi.ts

- Team 接口加 `area?: string | null`, `department?: string | null`
- StaffResponse 加 `areas?: Record<string, { name: string; description: string; icon: string }>`

### StaffPage.tsx

重写为三层结构：
1. Area 大标题（Cecelia / ZenithJoy）
2. Department 子标题（核心团队 / 运维部 / 新媒体 & 集成部）
3. Worker 卡片网格

## 验收标准

- [ ] 页面显示 Cecelia / ZenithJoy 两个 Area 标题
- [ ] 每个 Area 下有 Department 分组
- [ ] Worker 卡片保持现有设计（名字、角色、模型、skill）
