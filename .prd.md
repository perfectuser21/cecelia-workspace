## PRD - 添加 perfect21 到 Vite allowedHosts 白名单

**需求来源**: 用户通过 Tailscale hostname (perfect21) 访问开发环境时被 Vite 拦截

**问题描述**:
访问 `http://perfect21:5212` 时，Vite 显示错误：
```
Blocked request. This host ("perfect21") is not allowed.
To allow this host, add "perfect21" to `server.allowedHosts` in vite.config.js.
```

**功能描述**:
在 `apps/dashboard/vite.config.ts` 中添加 "perfect21" 到 `server.allowedHosts` 数组，允许通过 Tailscale hostname 访问。

**实现方案**:
修改 `apps/dashboard/vite.config.ts` 文件：
```typescript
server: {
  allowedHosts: [
    'localhost',
    'perfect21',  // 添加 Tailscale hostname
  ],
}
```

**涉及文件**:
- `apps/dashboard/vite.config.ts` - 添加 perfect21 到 allowedHosts

**技术要点**:
1. Vite 的 allowedHosts 白名单控制哪些 hostname 可以访问开发服务器
2. Tailscale 使用 MagicDNS 提供内网 hostname 解析
3. 保持现有配置不变，只添加新 hostname

**成功标准**:
1. `http://perfect21:5211` 和 `http://perfect21:5212` 都能正常访问
2. 不影响现有 localhost 访问
3. npm run build 成功编译
4. 开发环境标识显示正常

**非目标**:
- 不修改其他 Vite 配置
- 不修改 Tailscale 配置
- 不影响生产环境部署
