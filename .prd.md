# PRD: 修复登录后跳转问题

## 问题描述
用户访问 `/features` 页面时未登录，被重定向到 `/login?redirect=/features`。但是登录成功后，用户被跳转到首页 `/` 而不是原本想访问的 `/features` 页面。

## 根本原因
飞书 OAuth 回调流程：
1. 用户访问 `/login?redirect=/features`
2. 用户扫码登录
3. 飞书回调到 `/login?code=xxx` ← **redirect 参数被覆盖了**
4. 代码尝试从 URL 读取 redirect 参数，但已经不存在了
5. 默认跳转到首页 `/`

## 解决方案
使用 localStorage 保存 redirect 参数：
1. 在 FeishuLogin 组件加载时，如果 URL 中有 redirect 参数，保存到 localStorage
2. 登录成功后，优先从 localStorage 读取 redirect 参数
3. 跳转后立即清除 localStorage 中的 redirect 参数

## 验收标准
1. 用户访问 `/features` 未登录时，URL 显示 `/login?redirect=/features`
2. redirect 参数被保存到 localStorage
3. 用户扫码登录成功后，自动跳转到 `/features` 而不是首页
4. localStorage 中的 redirect 参数被清除
5. 没有 redirect 参数时，默认跳转到首页 `/`
