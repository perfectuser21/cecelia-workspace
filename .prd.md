## PRD - OKR 页面去重：提取共享组件

**需求来源**: 前端页面整理，OKRPage 和 OrchestratorPage 有重复的内联组件

**功能描述**:

OKRPage（战略视图）和 OrchestratorPage（运营仪表盘）虽然都展示 OKR 数据，但 UI 和交互模式完全不同，不适合强行合并。本次只提取两者重复定义的内联组件到 shared 目录：

1. **ProgressBar** — 两个页面都自定义了进度条
2. **StatusIcon** — 两个页面都根据 status 返回不同图标
3. **PriorityBadge** — 两个页面都根据 priority 返回不同颜色标签

提取到 `shared/components/` 后，两个页面改为从 shared 导入。

**涉及文件**:
- 创建：`apps/core/features/shared/components/ProgressBar.tsx`
- 创建：`apps/core/features/shared/components/StatusIcon.tsx`
- 创建：`apps/core/features/shared/components/PriorityBadge.tsx`
- 修改：`apps/core/features/planning/pages/OKRPage.tsx`（删除内联组件，改用 shared）
- 修改：`apps/core/features/execution/pages/OrchestratorPage.tsx`（删除内联函数，改用 shared）

**成功标准**:
1. OKRPage 和 OrchestratorPage 不再各自定义 ProgressBar/StatusIcon/PriorityBadge
2. 两个页面从 shared 导入这些组件
3. Vite 构建通过
4. 页面外观不变

**非目标**:
- 不合并 OKRPage 和 OrchestratorPage — 它们是不同用途的页面
- 不改变任何页面的功能或交互逻辑
