## PRD - 前端页面去重：合并 Cecelia 监控 + 统一 Home 页面

**需求来源**: 用户发现前端页面特别乱，存在大量重复/重叠的页面，要求整理去重

**功能描述**:

本次聚焦最明确的两组重复，不做大架构调整：

### 1. 合并 4 个 Cecelia/Brain 监控页 → 1 个

当前有 4 个页面展示几乎相同的 Brain 状态数据：
- `brain/pages/CeceliaOverview.tsx` (422行) — seats + task queue + process details
- `execution/pages/CeceliaOverview.tsx` (475行) — 同上 + timeline + cluster status
- `brain/pages/SeatsStatus.tsx` (311行) — 仅 seats 部分
- 路由指向重复

**方案**: 保留 `execution/CeceliaOverview.tsx`（最完整），删除 `brain/CeceliaOverview.tsx` 和 `brain/SeatsStatus.tsx`。更新 brain feature 路由指向 execution 的 CeceliaOverview。

### 2. 统一 5 个 Home 导航页 → 1 个通用组件

当前 5 个 Home 页面结构完全一样（导航卡片）：
- `today/pages/TodayHome.tsx` (49行)
- `work/pages/WorkHome.tsx` (54行)
- `knowledge/pages/KnowledgeHome.tsx` (48行)
- `system-hub/pages/SystemHome.tsx` (108行)
- `dashboard/pages/DashboardHome.tsx` (73行)

**方案**: 创建 `shared/pages/GenericHome.tsx` 通用组件，接收配置参数（title, cards），5 个 Home 页面改为调用通用组件传入不同配置。

**涉及文件**:
- 删除：`apps/core/features/brain/pages/CeceliaOverview.tsx`
- 删除：`apps/core/features/brain/pages/SeatsStatus.tsx`
- 修改：`apps/core/features/brain/index.ts`（路由改为指向 execution/CeceliaOverview）
- 创建：`apps/core/features/shared/pages/GenericHome.tsx`（通用 Home 组件）
- 修改：`apps/core/features/today/pages/TodayHome.tsx`（改用 GenericHome）
- 修改：`apps/core/features/work/pages/WorkHome.tsx`（改用 GenericHome）
- 修改：`apps/core/features/knowledge/pages/KnowledgeHome.tsx`（改用 GenericHome）
- 修改：`apps/core/features/system-hub/pages/SystemHome.tsx`（改用 GenericHome）
- 修改：`apps/core/features/dashboard/pages/DashboardHome.tsx`（改用 GenericHome）

**成功标准**:
1. brain feature 的 Cecelia 路由正常工作，展示 execution/CeceliaOverview
2. 5 个 Home 页面使用统一的 GenericHome 组件，外观不变
3. Vite 构建通过，无 TypeScript 错误
4. 减少约 800+ 行重复代码

**非目标**:
- 不合并 OKR 页面（OKRPage vs OrchestratorPage）— 留到下期
- 不合并 CommandCenter/PlannerOverview/PanoramaV3 — 功能差异较大
- 不动 Autopilot 旧页面 — 独立实例，暂不迁移
- 不改任何页面的视觉/功能逻辑
