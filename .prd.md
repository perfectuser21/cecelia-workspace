## PRD - Core 前端 5 入口 + 页面归位

**需求来源**: 用户需求 - 侧边栏从 6 条线简化为 5 个顶级入口

**功能描述**:
重写 Core 前端侧边栏导航，从当前 6 个 navGroup（Dashboard/Daily/GTD/Content/AI Dev/Brain）改为 5 个顶级入口（Dashboard/Today/Work/Knowledge/System）。所有 38 个现有页面不删除，全部归位到 5 个入口下。二级页面在页面内部导航（tabs/cards），不在全局侧边栏展开。

- `apps/core/features/today/index.ts` — 新建 manifest
- `apps/core/features/today/pages/TodayHome.tsx` — 新建首页（tabs: Today/Scheduler/Tasks）
- `apps/core/features/work/index.ts` — 新建 manifest
- `apps/core/features/work/pages/WorkHome.tsx` — 新建首页（card grid: Projects/Tasks/OKR/Roadmap/DevTasks/Features/Panorama/Whiteboard）
- `apps/core/features/knowledge/index.ts` — 重写 manifest
- `apps/core/features/knowledge/pages/KnowledgeHome.tsx` — 新建首页（cards: Content Studio/Super Brain）
- `apps/core/features/system-hub/index.ts` — 新建 manifest
- `apps/core/features/system-hub/pages/SystemHome.tsx` — 新建首页（grouped cards: Cecelia/Engine/N8N/Workers/Monitoring/Panorama）
- 删除旧 manifest：`daily/index.ts`, `gtd/index.ts`, `content/index.ts`, `aidev/index.ts`, `brain/index.ts`

**成功标准**:
1. `npm run build` 通过，无编译错误
2. 侧边栏只显示 5 个入口：Dashboard / Today / Work / Knowledge / System
3. 点击每个入口进入对应 Home 页面
4. 每个 Home 页面有 tabs 或 card grid 导航，点击可跳转到子页面
5. 所有上表中的路由可正常访问
6. 旧路由（/tasks, /projects, /cecelia 等）通过 redirect 仍然可达
7. 无 console error（Chrome DevTools，正常导航操作）

**非目标**:
- 不修改任何现有页面组件的内容（只改路由和 manifest）
- 不删除 pages/ 目录下的页面文件
- 不增强 DashboardHome（后续任务）
- 不重做 TodayView（后续任务）
