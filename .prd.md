# PRD: Fix AccountsList Login Route

## 功能描述
修复 AccountsList 页面中登录按钮导航路由与 LoginPage 路由定义不匹配的问题。

## 背景
- AccountCard 组件的 onLogin 回调只传递内部 `id`
- AccountsList 的 handleLogin 导航到 `/accounts/${id}/login`
- 但 LoginPage 路由定义为 `/login/:platform/:accountId`
- 导致点击"重新登录"按钮时路由不匹配

## 成功标准
1. AccountCard 的 onLogin 传递 platform 和 accountId
2. AccountsList 导航到正确路由 `/login/${platform}/${accountId}`
3. npm run build 通过
4. 点击登录按钮能正确跳转到 LoginPage
