## PRD - 移除 core/dev-core 实例区分，简化为只使用 localhost

**需求来源**: 用户要求 "我们已经没有 core https://core.zenjoymedia.media/work/okr 以及 dev-core 你现在把关于DevCore和Core这些全部都删掉。然后全部拿或者拿本地去取代。"

**功能描述**:
移除前端代码中所有关于 core/dev-core 实例的检测和区分逻辑，简化为统一使用 localhost。当前系统通过检测域名（core.zenjoymedia.media, dev-core.zenjoymedia.media）和端口（5211, 5212）来区分不同实例，这套逻辑已经不再需要，应该完全移除。

**涉及文件**:
- `apps/core/features/registry.ts` - 移除 detectInstance() 函数中的域名和端口检测逻辑
- `apps/core/features/types.ts` - 简化或移除 InstanceType 类型定义
- `apps/dashboard/vite.config.ts` - 移除 allowedHosts 中的 dev-core 和 core 域名
- `apps/dashboard/src/contexts/InstanceContext.tsx` - 已经硬编码 isCore = true，无需修改
- 其他可能引用 instance detection 的文件

**技术要点**:
1. registry.ts 的 detectInstance() 始终返回 'core'
2. 移除所有域名相关的判断逻辑
3. vite.config.ts 只保留 localhost
4. 确保所有 API 调用使用 localhost 而不是远程域名

**成功标准**:
1. `npm run build` 成功编译
2. 前端访问 localhost:5212 正常工作
3. 所有 API 请求正确代理到 localhost:5211
4. 代码中不再有 dev-core/core.zenjoymedia.media 的引用（除了注释和文档）
5. InstanceType 类型简化或移除，不再有 'autopilot' 等多余选项

**非目标**:
- 不修改后端 API 代码
- 不修改部署脚本（deploy/deploy-hk.sh 等）
- 不修改 Cloudflare Tunnel 配置
- 不修改 nginx 配置
