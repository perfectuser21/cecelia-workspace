# PRD: Organ Unification Phase 2 - Skills 执行归一

## 背景

Phase 1 已完成聚合层强化。Phase 2 目标是将纯执行类 Skills 迁移到 cecelia-workflows，建立统一的执行能力中心。

## 迁移清单

### P0 立即迁移（内容创作系列）

| Skill | 源位置 | 目标位置 |
|-------|--------|---------|
| `/create` | zenithjoy-creator/skills/create/ | cecelia-workflows/skills/content-creator/ |
| `/analyze` | zenithjoy-creator/skills/analyze/ | cecelia-workflows/skills/content-analyzer/ |
| `/rewrite` | zenithjoy-creator/skills/rewrite/ | cecelia-workflows/skills/content-rewriter/ |

### P1 迅速迁移（自媒体+配图）

| Skill | 源位置 | 目标位置 |
|-------|--------|---------|
| `/platform-scraper` | ~/.claude/skills/platform-scraper/ | cecelia-workflows/skills/platform-scraper/ |
| `/image-gen-workflow` | ~/.claude/skills/image-gen-workflow/ | cecelia-workflows/skills/image-gen-workflow/ |

### 不迁移（决策/流程类）

- `/dev`, `/gate`, `/qa`, `/audit`, `/assurance` - 保留在 zenithjoy-engine
- `/credentials`, `/github-protection` - 保留在本地（敏感/系统级）

## 成功标准

- [ ] 5 个 Skills 迁移到 cecelia-workflows/skills/
- [ ] 创建 symlink 保持向后兼容（2 周过渡期）
- [ ] 迁移后 Skills 功能正常
- [ ] 文档更新：新路径、调用方式

## 实现步骤

### 1. 在 cecelia-workflows 创建目录结构

```bash
mkdir -p cecelia-workflows/skills/{content-creator,content-analyzer,content-rewriter,platform-scraper,image-gen-workflow}
```

### 2. 复制 Skill 文件

```bash
# P0: 内容创作系列
cp -r zenithjoy-creator/skills/create/* cecelia-workflows/skills/content-creator/
cp -r zenithjoy-creator/skills/analyze/* cecelia-workflows/skills/content-analyzer/
cp -r zenithjoy-creator/skills/rewrite/* cecelia-workflows/skills/content-rewriter/

# P1: 自媒体+配图
cp -r ~/.claude/skills/platform-scraper/* cecelia-workflows/skills/platform-scraper/
cp -r ~/.claude/skills/image-gen-workflow/* cecelia-workflows/skills/image-gen-workflow/
```

### 3. 创建 Symlinks（兼容旧路径）

```bash
# 旧路径指向新位置
ln -sf /home/xx/dev/cecelia-workflows/skills/content-creator ~/.claude/skills/create
ln -sf /home/xx/dev/cecelia-workflows/skills/content-analyzer ~/.claude/skills/analyze
ln -sf /home/xx/dev/cecelia-workflows/skills/content-rewriter ~/.claude/skills/rewrite
```

### 4. 更新 SKILL.md 中的路径引用

检查每个 SKILL.md，更新内部路径引用。

### 5. 验证测试

```bash
# 验证 symlink
ls -la ~/.claude/skills/create
ls -la ~/.claude/skills/analyze

# 验证功能（调用 skill）
# /create test
# /analyze test
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| Symlink 损坏 | 验证脚本检查 |
| 路径硬编码 | 使用相对路径 |
| 功能中断 | 保留原目录 2 周 |
